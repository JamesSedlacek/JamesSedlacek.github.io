Features

Sorting Animation

I used UIAnimation, UITransition, and a Timer to move multiple subview images that were created programmatically.

Dark Mode

I created different light/dark appearance image assets for the bars, background, font color, and button colors. Whenever the switch is tapped for dark mode, every single screen is changed into dark mode.

PickerView & Google AdMob

There's quite a few moving parts with this feature. My favorite thing to work on in this project was the interstitial ad that plays after the 'done' button is tapped.

CollectionView

I wanted to work with collection views somehow in this project. I was able to create some cards that display the different cases for efficiency for each algorithm.

UIAnimation Splash Screen

One of my favorite things to work on in this app was this feature. I used UITransition, UIView.Animate, and a completion handler. I also had to get the iPhone's screen width to calculate the width of each bar.

Why?

Inspiration

Clement's sorting algorithm visualizer project that helped him get into Google was the main source of inspiration for this project.

I also wanted to practice using everything I was learning in my Udemy course.

Challenges

Calculating the Bar's Maximum Height

During the final testing of the project, I found it to be very complicated to solve the problem of making everything look right on all screen sizes. This is how I solved the problem of finding the maximum height for the bars. A problem like this, you can't just google it, you have to think critically.

To start, I used the stackView (the container that the buttons are in) because it has the greatest Y coordinate. Then I subtracted it's height. Then I subtracted the algoLabel Y coordinate. Finally, I subtracted the spaces in between the bars and the buttons/algoLabel (the barMargin). This will leave you with the maximum height of each bar.

*Disclaimer* The Y-axis starts at zero at the top of the iPhone and increases going downward.
